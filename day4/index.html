üîå API Endpoints

1. Get All Employees (In-Memory)

GET http://localhost:8080/employee

Description: Returns list of all employees from in-memory storage

Response:

[
  { "id": 1, "name": "Amit", "department": "IT", "salary": 24000 },
  { "id": 2, "name": "Ramesh", "department": "HR", "salary": 21000 }
]

2. Get All Employees (From Database)

GET http://localhost:8080/employeedb

Description: Returns all employees from MySQL database using native SQL query

3. Add New Employee

POST http://localhost:8080/employee

Request Body (JSON):

{
  "id": 4,
  "name": "Kiran",
  "department": "IT",
  "salary": 32000
}

Test in Postman/Thunder Client:

  • Set method to POST
  • Set Content-Type to application/json
  • Add JSON body

4. Update Employee Salary

PUT http://localhost:8080/employee/2

Request Body (JSON):

{
  "salary": 25000
}

Description: Updates salary of employee with specified ID

5. Delete Employee

DELETE http://localhost:8080/employee/2

Response:

"Employee record delete successfully.."

üîß Key Components Explained

1. Employee.java (POJO)

Plain Old Java Object used for in-memory operations with getters, setters, and toString method.

2. EmployeeEntity.java (JPA Entity)

Mapped to employees table in database with JPA annotations:

  • @Entity - Marks class as JPA entity
  • @Table(name = "employees") - Maps to database table
  • @Id - Primary key field
  • @GeneratedValue - Auto-increment ID

3. EmployeeRepository.java

Interface extending JpaRepository with custom native SQL queries:

  • getAllEmployeesNative() - SELECT * FROM employees
  • findByDepartmentNative() - Filter by department

4. EmployeeController.java

REST controller handling HTTP requests with annotations:

  • @RestController - Marks as REST controller
  • @GetMapping - Handle GET requests
  • @PostMapping - Handle POST requests
  • @PutMapping - Handle PUT requests
  • @DeleteMapping - Handle DELETE requests
  • @PathVariable - Extract value from URL
  • @RequestBody - Parse JSON from request body

‚ùì Common Issues & Solutions

Issue 1: Application won't start

Solution: Check if port 8080 is already in use. Close other applications or change port in application.properties

Issue 2: Database connection failed

Solution: Verify MySQL is running, database exists, and credentials are correct

Issue 3: Employee not found

Solution: Check if employee ID exists in the list. Remember in-memory and database storage are separate

Issue 4: JSON parsing error

Solution: Ensure Content-Type header is set to application/json in Postman
