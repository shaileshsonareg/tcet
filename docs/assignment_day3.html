1. Project Overview

The Employee Management System is a full-stack web application that enables users to perform Create, Read, Update, and Delete (CRUD) operations on employee records. This mini project demonstrates the integration of a modern frontend interface with a RESTful Spring Boot backend.

4. Backend Requirements (Spring Boot)

4.1 REST API Endpoints

Your Spring Boot application must implement the following RESTful endpoints:

GET /employee

Retrieve all employees (returns ArrayList<Employee>)

POST /employee

Create a new employee (accepts Employee JSON)

PUT /employee/{id}

Update an existing employee by ID

DELETE /employee/{id}

Delete an employee by ID

4.2 CORS Configuration

Enable CORS in your Spring Boot controller to allow frontend communication:

@CrossOrigin(origins = "http://localhost:*")
@RestController
@RequestMapping("/employee")
public class EmployeeController {
    // Your REST endpoints here
}

6. Functional Workflow

6.1 Application Flow

  1. Page Load: Application automatically fetches all employees from backend
  2. Display Data: Employees are rendered in a table format
  3. Add Operation:
    • User clicks "Add New Employee" button
    • Modal popup opens with empty form
    • User fills in employee details
    • On submit, POST request sent to backend
    • Modal closes and list refreshes
  4. Edit Operation:
    • User clicks "Edit" button on specific employee row
    • Modal popup opens with pre-filled data
    • User modifies details
    • On submit, PUT request sent to backend
    • Modal closes and list refreshes
  5. Delete Operation:
    • User clicks "Delete" button on specific employee row
    • Confirmation dialog appears with employee name
    • If confirmed, DELETE request sent to backend
    • Dialog closes and list refreshes

12. Common Issues and Solutions

Issue: CORS Error

Solution: Add @CrossOrigin annotation to your Spring Boot controller

Issue: 404 Not Found

Solution: Verify backend is running and API_URL is correct

Issue: Data not refreshing

Solution: Ensure loadEmployees() is called after each CRUD operation

Issue: Modal not closing

Solution: Check closeModal() function is properly called
